Machine Clients are non-human identities used by servers, workloads, and automation to access private resources.

Machine clients do **not** authenticate with username/password. They authenticate using a generated client **`id`** and **`secret`**.

## What This Page Is For

Use this page to manage machine identities for workloads that cannot use interactive user login.

Typical actions include:

- Creating new machine clients
- Reviewing existing client entries
- Finding a client by name/environment
- Removing or rotating clients that are no longer needed

## Use Cases

- Backend services connecting to private APIs or databases
- CI/CD jobs requiring internal network access
- Scheduled automation tasks running headless

## Lifecycle

1. Create a machine client.
2. Store credentials securely (`id` and `secret`).
3. Install and run the client on target infrastructure.
4. Monitor health and rotate credentials on schedule.

## Operational Guidance

- Use separate machine clients per workload and environment.
- Avoid sharing one machine credential across unrelated systems.
- Prefer short operational ownership chains (each key has a clear owner/team).
- Revoke unused machine clients during regular access reviews.
