Invitations is the **Open Invitations** page. It shows outstanding invites for users who have not joined yet.

## What This Page Is For

- View pending invites in a table
- Search invitations by email
- Check expiration and assigned role
- Refresh the list
- Regenerate invitation links for existing entries

## Best Practices

- Keep invite lists clean by revoking stale invitations.
- Verify the correct email and intended role before sending.
- Reissue invites instead of leaving expired or invalid ones pending.

## Table Fields

- **Email**: The invited user email.
- **Expires At**: When the invitation link becomes invalid.
- **Role**: The role the invited user will receive on acceptance (for example, `Member`).

## Actions

- **Refresh**: Reload the invitation list.
- **Regenerate**: Create a new invitation link/token for the selected invite when the user cannot use the previous one.
- **More Actions (`...`)**: Open row actions, including **Remove Invitation**.

## Troubleshooting

If users cannot complete onboarding:

- Confirm the invitation is still active.
- Confirm they are signing in with the same email/account that was invited.
- Revoke and resend the invitation if it is stale or invalid.
- Check [Approval Requests](/manage/access-control/approvals) if additional approval is required.
- If SMTP is not configured, Pangolin will present an invite link in the UI. Copy that link and send it to the user manually.
