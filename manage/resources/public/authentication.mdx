Use the **Authentication** tab on a public resource to control who can access it and which extra auth methods are enabled.

## Access Controls

This section controls baseline access using platform identity.

- **Use Platform SSO**: Require Pangolin sign-in before resource access.
- **Roles**: Allow selected roles to access the resource.
- **Users**: Allow specific users to access the resource.

Admins can always access the resource.

Use **Save Access Controls** after changes.

## Authentication Methods

This section allows additional auth methods on top of (or alongside) your access controls.

- **Password Protection** (`Add Password`)
- **PIN Code Protection** (`Add PIN Code`)
- **Header Authentication** (`Add Header Auth`)

Use this section when you need lightweight shared protection or programmatic header-based access patterns.

## One-time Passwords

Enable email-based one-time passwords with:

- **Email Whitelist**

When enabled, users must provide an allowed email to receive a one-time code before access is granted.

Use **Save Whitelist** after updates.
