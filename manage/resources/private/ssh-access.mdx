Use **SSH Access** to configure how SSH authentication is handled for this private resource.

## Availability

In some deployments, SSH resource features may require an eligible plan or edition.  
If unavailable, the UI shows an upgrade prompt.

## SSH Auth Daemon Location

Choose where the SSH authentication daemon runs:

- **On Site**: Auth daemon runs on the site connector host (Newt).
- **Remote Host**: Auth daemon runs on the resource destination host.

This controls where SSH key signing and PAM authentication handling occurs for the resource.
