Pangolin Cloud is the fastest way to start using Pangolin. It gives you a hosted control plane for identity, policy, DNS, certificates, backups, and product updates without requiring you to run the full management stack yourself.

You can stay fully managed, or you can attach self-hosted remote nodes to keep tunnels, SSL termination, and traffic on infrastructure you control. That makes Cloud a strong default for teams that want simpler operations without giving up deployment flexibility.

## Why Teams Start with Cloud

- **Fastest time to value**: Create an account and start configuring Pangolin immediately without deploying the full control plane first.
- **Less operational overhead**: Pangolin handles the control plane, backups, and ongoing dashboard updates for you.
- **Stronger team onboarding**: Invite users, manage access, and expand the deployment from a shared hosted dashboard.
- **Hybrid deployment path**: Attach remote nodes when you want traffic and SSL termination to stay on servers you control.

## Use Cloud If You Want To

- Start quickly without maintaining the full Pangolin control plane
- Keep a path open to higher-tier Cloud features like custom auth pages, device approvals, and organization identity providers as your deployment grows
- Let Pangolin handle DNS, certificate management, and backups
- Keep a path open to self-hosted remote nodes later

## What Cloud Does Not Replace

Pangolin Cloud does not force you into a single deployment model. If your services live on your own networks, you still connect them with sites and resources. If you want more infrastructure control, you can deploy remote nodes and keep traffic on your own servers.
